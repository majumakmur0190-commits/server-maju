Audit Folder API

Dokumentasi API Aplikasi Maju

Dokumen ini dibuat dari pembacaan langsung semua file di folder api. Isinya mencakup endpoint utama, file utilitas, file cetak PDF, file debug, cara kirim data, dan bentuk response aktual.

Base URL lokal: /maju/api/. Mayoritas endpoint menerima JSON melalui Content-Type: application/json, tetapi ada beberapa file lama yang menerima form biasa atau bahkan mengeluarkan HTML/PDF.

Cara Pakai

Format Request Umum

  • Header paling sering dipakai: Content-Type: application/json.
  • Data filter list dikirim lewat query string seperti ?id=, ?search=, ?page=, ?limit=.
  • Data simpan atau update umumnya dikirim dalam JSON body.
  • Banyak endpoint sudah menyiapkan OPTIONS untuk kebutuhan CORS.
Contoh Fetch

Kirim JSON ke API

fetch('/maju/api/barang.php', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    barcode: "899001",
    nama_barang: "KOPI BUBUK",
    kategori_id: 2,
    harga_hna: 12000,
    stok: 15,
    satuan: "PCS"
  })
}).then(r => r.json()).then(console.log);
Contoh cURL

Login API

curl -X POST http://localhost/maju/api/login.php ^
  -H "Content-Type: application/json" ^
  -d "{\"username\":\"admin\",\"password\":\"12345\"}"

Endpoint Inti Aplikasi

Master Barang

api/barang.php

GET POST PUT DELETE

CRUD barang aktif. Semua response dibungkus dengan format {status, data/message}.

GET /maju/api/barang.php?id=5 GET /maju/api/barang.php?search=kopi POST /maju/api/barang.php PUT /maju/api/barang.php?id=5 DELETE /maju/api/barang.php?id=5

Cara Kirim Data

  • GET: query string id atau search.
  • POST/PUT: JSON body dengan field barcode, nama_barang, kategori_id, harga_hna, stok, satuan.
  • DELETE: cukup kirim ?id=....
{
  "barcode": "899001",
  "nama_barang": "KOPI BUBUK",
  "kategori_id": 2,
  "harga_hna": 12000,
  "stok": 15,
  "satuan": "PCS"
}

Contoh Response

{
  "status": "success",
  "data": [
    {
      "barang_id": 5,
      "barcode": "899001",
      "kategori_id": 2,
      "harga_hna": 12000,
      "stok": 15,
      "aktif": 1
    }
  ]
}

DELETE adalah soft delete: aktif diubah menjadi 0.

Catatan Penting Dari Audit Source

Perbedaan Format Request

  • Mayoritas endpoint CRUD memakai JSON body.
  • login.php, barcode.php, dan inventor.php masih bisa menerima form biasa.
  • print*.php dan invoice.php hanya butuh query string id.

Perbedaan Format Response

  • Banyak endpoint memakai format {status, data}.
  • gambar.php kadang mengembalikan array langsung atau {success:true}.
  • printer-list.php mengembalikan array murni.
  • print*.php menghasilkan PDF dan order.php menghasilkan HTML.

Temuan Khusus

  • barcode.php?action=get-all-barang tampak salah query dan sangat mungkin gagal.
  • sycn.php kemungkinan salah eja nama file, tetapi memang itu file yang dipakai sekarang.
  • pass.php berisiko jika bisa diakses publik.
  • gambar.php membutuhkan extension GD dan izin tulis ke folder api/upload.
